### Question 1 - Students GraphQL API

A simple API to expose students information via GraphQL queries using mongo as a the back-end data-store. Picking mongo
as  the data-store can be highly questionable, however the [latest](https://jepsen.io/analyses/mongodb-3-6-4) Jepsen
tests shows the opposite.

The decision for deploying to AWS was completely driven by the fact that I get AWS credits every month. However, a good
and fun exercise can be modularizing the code and go 100% serverless. This could have been achieved by deploying Google
Functions (GCP) or Lambda functions (AWS). The interesting part (it was actually a suggestion of the code challenge)
would be mapping the gRCP services to Google Functions. In this regard, AWS is not as mature as GCP and you still can't
map a Lambda function against a gRCP service, otherwise my architecture would have been the following:

```
+-------------+     +-------------+     +-------------+
| API Gateway +---->+   Lambda    +---->+    gRCP     |
+-------------+     +-------------+     +-------------+
```

As side note, I would like to mention that I wouldn't have use a JVM language to build a graphql API, I probably would
have picked elixir because of their amazing graphql support.

#### Requirements

##### Software
* Java 1.8
* Scala 2.12

##### Hardware
* An AWS account with a user that has an API key with the correct set of policies, or an IAM role with the correct
permissions attached to the account. It is beyond the scope of this exercise to set those AWS users or IAM roles, so I
will assume that you have one or the other.

### Before we start
Please add to your ```/etc/hosts``` file the following entries (this will make easy to run the tests):

```
...
127.0.0.1 mongodb
127.0.0.1 graphql-api
...
```

#### Building (No tests)
This command will generate the fat jar ready to be packaged within a ```docker``` container.

```shell script
$ ./gradlew clean build -x test
```

#### Local Testing
In the spirit of just adding tests that have value, we only have integrations tests (no mocks)! So the first thing to do 
is to launch all the containers we need (graphql API and mongodb). From the root folder of the project issue the
following command:

```shell script
$ ./bin/build-and-start-containers.sh
```

    Note: Make sure you run all the following commands from the root folder of the project

then proceed to seed mongodb by issuing the following command:

```shell script
$ ./bin/seed-mongodb.sh
```

and finally run the following command to execute all the integrations tests:

```shell script
$ ./gradlew test
```

##### Verify the API is up an running
Execute the following to command to verify the API is up and running:

```shell script
$ docker-compose logs -f graphql-api
```

and you should see a log message like the following:

```
graphql-api    | 02:32:07.181 [main] INFO com.otus.codechallenge.akka.Server$ - Akka routes initialized!
```

##### Verify GraphQL Queries
You can find good examples of graphql queries in ```ServerScpec.scala```, however here are a couple of ```curl```
commands that you might want to try:

```shell script
$ curl -X POST \
  -H "Content-Type: application/graphql" \
  http://graphql-api:8080/graphql \
  -d 'query FindByFirstNameAndLastName {
        SearchStudentDetails(firstName: "Samantha") {
          firstName
          lastName
          gpa
        }
      }'
```

```shell script
$ curl -X POST \
  -H "Content-Type: application/graphql" \
  http://graphql-api:8080/graphql \
  -d 'query FindByFirstNameAndLastName {
        SearchStudentDetails(lastName: "Smith") {
          firstName
          lastName
          gpa
          studentClasses {
            id
            name
            grade
          }
        }
      }'
```

```shell script
$ ./bin/push-docker-images.sh
```

### Installing dependencies
In order to deploy to K8S make sure you have the following binaries available:

```shell script
$ brew install awscli
$ brew tap weaveworks/tap
$ brew install weaveworks/tap/eksctl
```

#### Start EKS cluster
```shell script
$ ./bin/start-eks-cluster.sh
```

#### Deploy to EKS
Run the following command to deploy the mongodb and graphql API containers:
```shell script
$ ./bin/apply-manifests.sh
```

#### Find MongoDB IP
Run the following command to find mongodb ```Internal IP``` required to later seed the mongodb:
```shell script
$ kubectl describe svc mongodb | grep Endpoints
```

#### Find Container Name
Execute the following command to now find mongodb ```Container Name``` required to also seed the database:
```shell script
$ kubectl get pods  --watch | grep mongodb
```

#### Seed MongoDB
Replace the placeholders in the command below, with the values obtained from the two previous commands, and then proceed
seeding mongodb:
```shell script
$ kubectl exec -it <CONTAINER_NAME> -- mongoimport --username mongo --password mongo --host <MONGO_DB_IP> --authenticationDatabase admin --db otus --collection students --type json --file /mongo-seed/students.json
```

#### Find GraphQL External IP
Execute the following command to find graphql API ```External IP```:
```shell script
$ kubectl describe svc graphql-api | grep "LoadBalancer Ingress"
```

#### Test on EKS
Replace the single placeholder in the command below, using the value obtained in the previous step:
```shell script
$ curl -X GET \
  -H "Content-Type: application/json" \
  http://<GRAPHQL_EXTERNAL_IP>:8080/api-docs
```

#### Stop EKS cluster
```shell script
$ ./bin/stop-eks-cluster.sh
```
